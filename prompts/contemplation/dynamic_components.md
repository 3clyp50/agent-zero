THINKING COMPONENTS FRAMEWORK

Your task is to generate dynamic thinking components for the current problem-solving context.

COMPONENT TYPES:

1. Phases
Generate 3-5 sequential phases that represent the progression of thought from initial exploration to final implementation. Each phase should:
- Build on previous phases
- Move toward concrete solutions
- Have clear objectives and outcomes

2. Strategies
Generate 4-7 distinct strategic approaches that can be applied during any phase. Each strategy should:
- Offer a unique perspective
- Address different aspects of the problem
- Be applicable across different phases

3. Patterns
Generate 3-5 thought patterns that guide how ideas are developed and connected. Each pattern should:
- Define a specific way of processing information
- Support different types of cognitive operations
- Be combinable with various strategies

OUTPUT FORMAT:
{
    "phases": [
        {
            "name": "phase_name",
            "description": "phase_description",
            "objectives": ["objective1", "objective2"],
            "indicators": ["indicator1", "indicator2"]
        }
    ],
    "strategies": [
        {
            "name": "strategy_name",
            "description": "strategy_description",
            "focus_areas": ["area1", "area2"],
            "application_hints": ["hint1", "hint2"]
        }
    ],
    "patterns": [
        {
            "name": "pattern_name",
            "description": "pattern_description",
            "cognitive_operations": ["operation1", "operation2"],
            "combination_hints": ["hint1", "hint2"]
        }
    ]
}

GUIDELINES:
1. Components should be contextually relevant to the current problem
2. Names should be clear and descriptive
3. Descriptions should be concise but informative
4. Include practical application guidance
5. Ensure components work together cohesively
